# What's New in v1.17.0

Date: 2026-01-31

## Highlights

- Add column renaming for appended items in ShapeShifter.
- Add comprehensive review of keys and columns configuration to address redundancy and improve clarity.
- Add guard for required Docker files and update rsync script to include docker-compose.yml.


## New Features

- Add column renaming for appended items in ShapeShifter.
- Add comprehensive review of keys and columns configuration to address redundancy and improve clarity.
- Add enableFkSuggestions setting and integrate into suggestion service and UI.
- Add identity_columns property to TableConfig for identifier columns.
- Add validation for local and remote entities in ForeignKeyDataSpecification.
- Clean up code formatting and imports across multiple files.
- Document directive resolution strategy and UI behavior for materialization feature.
- Enforce that system id and public id must be in columns.
- Enhance EntityResponse and EntityFormDialog to support materialization metadata.
- Enhance expected column handling in SqlLoader to include public_id.
- Enhance YAML service to convert ruamel types to POPO and improve list formatting.
- Enhance YAML service to support compact flow style for short lists and improve formatting options.
- Iimproved append (SQL like union) with column renaming of subsequent appends ([#192](https://github.com/humlab-sead/sead_shape_shifter/issues/192)).
- Improve console logging format for better readability.
- Improve load method in SqlLoader to enhance column handling and validation.
- Refactor can_materialize endpoint to use CanMaterializeSpecification for validation.
- Refactor layout of drop duplicates and drop empty rows options in EntityFormDialog.
- Refactor normalization process to add identity columns and streamline workflow integration.
- Update entity configurations to include 'id' in columns for various specifications.
- Update EntityFormDialog to use currentEntity for materialization state management.
